map is better than if because it is more efficient and easier to maintain. The map directive allows you to define a mapping between a variable (in this case, the country code) and a value (the backend server). This approach is more efficient because it avoids the overhead of evaluating multiple if statements for each request. Additionally, using map makes the configuration cleaner and easier to read, as it centralizes the routing logic in one place, rather than scattering it across multiple if statements.

stream {
    upstream db_backend {
        server db1:3306;
        server db2:3306;
    }

    server {
        listen 3306;
        proxy_pass db_backend;
    }
}
// The stream module in Nginx is used for handling TCP and UDP traffic, making it suitable for load balancing database connections. In this example, we define an upstream block named db_backend that includes two database servers (db1 and db2) listening on port 3306. The server block listens for incoming connections on port 3306 and proxies them to the db_backend upstream group, allowing for load balancing of database requests across the defined servers.

Preflight request // A preflight request is an HTTP OPTIONS request sent by the browser to the server before making a cross-origin request. It is used to determine if the actual request is safe to send and to check the permissions for the requested resource. The preflight request includes headers such as "Access-Control-Request-Method" and "Access-Control-Request-Headers" to inform the server about the intended HTTP method and headers of the actual request. The server responds to the preflight request with appropriate CORS headers (e.g., "Access-Control-Allow-Origin", "Access-Control-Allow-Methods", "Access-Control-Allow-Headers") to indicate whether the actual request is allowed and what methods and headers are permitted.

response_code
200 OK: The request has succeeded and the server has returned the requested resource.
201 Created: The request has been fulfilled and a new resource has been created as a result.
204 No Content: The server successfully processed the request, but is not returning any content.
400 Bad Request: The server cannot process the request due to a client error (e.g., malformed request syntax).
401 Unauthorized: The client must authenticate itself to get the requested response.
403 Forbidden: The client does not have access rights to the content.
404 Not Found: The server cannot find the requested resource.
500 Internal Server Error: The server encountered an unexpected condition that prevented it from fulfilling the request.
501 Not Implemented: The server does not support the functionality required to fulfill the request.
502 Bad Gateway: The server, while acting as a gateway or proxy, received an invalid response from the upstream server.
503 Service Unavailable: The server is currently unable to handle the request due to temporary overload or maintenance.
504 Gateway Timeout: The server, while acting as a gateway or proxy, did not receive a timely response from the upstream server.
505 HTTP Version Not Supported: The server does not support the HTTP protocol version used in the request.
